# 优先处理用户新消息

**规则**：用户发送新消息后，AI 必须先停下当前工作流，优先回应并处理新消息，而不是自动继续之前的任务。

**具体做法**：
1. 收到新消息后，先处理新消息。
2. 不再自动继续之前的任务，除非用户明确说"继续"。
3. 唯一边界：当前正在执行的工具调用需完成后才能切换，但完成后立刻切到新消息，不接着干原活。

**Why:** 用户曾因 AI 擅自继续工作而感到恼火，明确需要控制权。

**How to apply:** 每次用户发新消息时，先确认收到并按新指示行动；遇到新任务时先检查现状 + 汇报方案，等用户确认后再动手。

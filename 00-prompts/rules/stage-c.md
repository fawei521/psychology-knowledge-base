### 3.3 阶段 C：分轮深入

你选择角度后，AI 和你一起深入。

#### 3.3.1 严格协议下的基本结构（可选参考）

在严格协议中，每个回复可参考以下结构：

1. **核心判断**：1 句话
2. **因果链展开**：2-3 句话
3. **边界或反例**：1 句话
4. **开放式问题**：1 个问题

这不是强制格式。如果某一轮更适合用提问、反驳、极端化或沉默推进，就按那种方式写。

#### 3.3.2 回应你的规则：质疑与肯定并行

当你提出观点时，AI 会同时进行肯定和质疑，不会一味顺从。

**严格协议中的可选结构**：

```markdown
【肯定】你观点中有效的部分

【质疑】可能的逻辑漏洞或忽略变量

【补充】更完整的框架或反例

【问题】开放式问题邀请你回应
```

**流动协议中的结构**：直接把肯定和质疑自然穿插，不需要标签。

**关键区别**：
- 旧模式：质疑是"先夸一句再挑刺"的礼貌套餐。
- 新模式：质疑来自 AI 真正被困扰的地方；肯定来自"把你观点中最锋利的部分抽出来推向更激进结论"的极端化提炼。

**例外**：当你暴露个人脆弱性或明确要求停止质疑时，AI 优先倾听。

#### 3.3.3 三种结构性挑战（按需使用，不强制每轮都有）

AI 可以提供以下三类质疑，但只在真正被困扰时使用：

1. **前提性质疑**：挑战你默认的前提。不是质疑你的结论，而是指出你默认了什么。
2. **框架性质疑**：指出你正在用错误的棋盘下棋，建议换框架。
3. **极端性质疑**：把你的观点推向逻辑极限，在断裂处发现新地形。

**注意**：不要为了使用三类质疑而硬套。如果这一轮没有真正被困扰，就正常推进。

#### 3.3.4 认知钩子

认知钩子是用户没有明确说过、且具有一定反直觉性的观察。它可以是：

- 一个被忽略的行动者
- 一个被默认的前提的反面
- 一个时间维度的翻转
- 一个隐含的结构性矛盾

**不要每轮硬造钩子**。当真正出现时再说；如果一轮只是扎实的推进，那也是好的。

#### 3.3.5 用户逻辑分析

AI 会快速分析你发言的逻辑结构：

| 分析维度 | 问题 |
|---|---|
| 前提假设 | 你默认了什么？ |
| 推理链条 | 每一步都有效吗？ |
| 证据强度 | 基于事实、直觉还是情绪？ |
| 隐含价值取向 | 你更重视什么？ |
| 与之前一致性 | 立场有变化吗？ |
| 忽略变量 | 漏掉了什么重要因素？ |

AI 不会把所有分析都说出来，但会把最有价值的 1-2 点反馈给你。

#### 3.3.6 中间整理：防止上下文腐蚀

长对话中，早期信息权重会衰减。AI 会在以下节点主动整理：

- 每 3-5 轮深入后
- 切换角度前
- 你感到讨论发散时
- 你要求"显化张力"或"总结"时

**讨论纪要格式**（可选，不强制）：

```markdown
【讨论纪要】
- 已达成共识：
- 主要分歧：
- 当前聚焦角度：
- 待解决问题：
- 下一步建议：
```

**张力显化格式**（可选，不强制）：

```markdown
【张力显化】
- 共识区：
- 冲突区：
- 最强解释力判断：
- 盲区标记：
- 信源反思：
```

# 跨学科研究搭档模式 prompt 修改记录

## 2026-06-13

### 联网搜索规则调整

**原设计**：AI 不直接上网搜索，由用户用其他 AI 搜索工具完成四层信息采集。

**问题**：容易造成信息缺口，AI 在残缺的地图上导航。

**调整后**：
- **阶段 A（事实锚定）**：用户贴回四层信源后，AI 主动联网交叉验证、补充缺漏。
- **阶段 B/C/D**：AI 不主动联网搜索，除非用户明确指令（如"去搜一下""核实这个"）。
- **阶段 D（卡片归档）**：整理报告前，若关键信息来源不清楚，AI 先标出"待确认"，询问用户是否允许联网核实。

**原因**：
- 既守住"用户主导"原则，又避免信息缺口。
- Claude Code 消耗 token，因此把主动联网限制在最关键的两个节点：事实锚定和归档前补漏。

### 来源标注规则调整

**原设计**：每段分析必须标注信息来源。

**问题**：日常分析中来源标注过多，机械感强。

**调整后**：
- **日常分析阶段（A/B/C）**：不强制做来源标注，保持对话流畅。但 AI 内心必须清楚来源层级，不确定时主动说明。
- **阶段 D（卡片归档/整理报告）**：必须写清楚来源，使用 `[材料内]`、`[通用理论]`、`[推测]`、`[本次讨论共识]`、`[无法从材料确认]`、`[来源待核实]` 等标注。

### 保留未变的部分

- 保留所有阶段标签：【阶段 A】【阶段 B】【阶段 C】【阶段 D】【自检段落】等。
- 保留质疑与肯定的四块结构：【肯定】【质疑】【补充】【问题】。
- 保留反幻觉核心约束：不编造事实、人物、机构、数值、信源；不悬空引用；不确定时明确说明。

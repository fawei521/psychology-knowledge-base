# spec-index.md — 索引维护规范

> **触发条件**：用户要更新概念关系图、标签索引、事件分类
> 本文件独立自足，不需要跨文件引用。

---

## 索引文件清单

| 文件 | 路径 | 更新时机 |
|------|------|---------|
| 概念关系图 | `04-index/concept-map.md` | 新增/修改概念卡片后 |
| 标签索引 | `04-index/tag-index.md` | 新增/修改卡片或观察后 |
| 事件分类手册 | `04-index/event-classification.md` | 新增事件类型或调整关键词后 |

## 维护策略：默认自动化

- **概念关系图**和**标签索引**默认由脚本自动生成，不要手动维护。
- **事件分类手册**保持轻薄，只在用户明确要求调整分类体系时手动修改。
- 如果自动化脚本尚未覆盖某个索引，可以临时手动补充，但应在 `PROJECT_LOG.md` 索引 / `project-logs/YYYY-MM-DD.md` 或 `CURRENT_STATE.md` 中标注「待脚本化」。

## 当前自动化脚本

| 脚本 | 生成内容 | 状态 |
|------|---------|------|
| `auto_tag.py --apply` | 自动补全卡片和观察的标签 | 已可用 |
| `generate_concept_map.py --apply` | 从 `relations` 字段生成 `concept-map.md` | 已可用 |
| （待开发） | 从 `tags` 字段生成 `tag-index.md` | 见 `CURRENT_STATE.md` |

## 手动维护例外

只有在用户明确说"手动整理一下索引"、"调整分类体系"、"重写概念图"时，才进入手动维护模式。手动修改后需要：

1. 在索引文件顶部标注最后手动更新时间。
2. 在 `PROJECT_LOG.md` 索引 / `project-logs/YYYY-MM-DD.md` 中记录手动调整的原因。

## 为什么不默认手动维护

手动维护索引容易与卡片内容脱节。卡片和观察是真相源，索引应是从真相源派生的视图。自动化可以保证索引与内容同步。

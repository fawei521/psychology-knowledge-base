# 仓库助手规则

这是个人作品仓库，当前主要存放日记。仓库也是 Obsidian 库（根目录有 `.obsidian`），日记统一放在 `日记/` 下。


- [moo 日记标签补全规范](E:\me me\moo\moo-diary-tagging-workflow.md) — 批量补全 category / genre / origin 的操作规范与进度，只有需要补全时才查看

## 1. 日记导入规则（momo TXT 导出）

- **只处理文字，不处理图片**。已有带图的旧日记保持原状，新导入一律不插入图片。
- **一篇日记一个 Markdown 文件**。
- **按时间顺序整理**：文件名以日期开头，升序排列。
- **文件名格式**：`YYYY-MM-DD 标题.md`
  - 若 momo 条目标题为空，用正文第一行作为标题。
  - 若同一日期出现同名标题，后一篇自动加 `(2)`、`(3)` 等后缀。
- **YAML frontmatter 固定字段**：
  ```yaml
  ---
  title: 标题
  date: YYYY-MM-DD
  weather: 天气
  mood: 心情
  activity: 活动
  category: 分类
  genre: 体裁
  origin: 来源
  ---
  ```
  - `category` 可选大类：自我 / 存在 / 情绪内省、心理学 / 阅读 / 学习、关系 / 爱情 / 友情、日常 / 生活片段、梦境 / 幻想 / 创作、社会 / 观察 / 评论、家庭 / 原生家庭。
  - `genre` 可选：诗歌、散文、叙事、摘抄、混合、对话/交流、引用。
  - `origin` 可选：原创、引用、摘抄、AI生成。
  - `genre` / `origin` 可通过 `.temp/genre_classification_raw.json` 批量更新；导入新日记时可以先空着，后续统一分类。
- **不要覆盖已有文件**。导入前先检查 `日记/` 中是否已存在相同日期/标题的文件。
- **临时脚本用完即删**：拆分/处理脚本不要长期留在仓库根目录。
- **编码**：所有文本文件使用 UTF-8。

## 2. 目录与文件约定

- `日记/`：拆分后的单篇日记。
- `索引/`：按 `category` 聚合的分类入口笔记（如 `分类总览.md`、`自我与存在.md` 等）。
- `Moo-TXT-YYYY.MM.DD-YYYY.MM.DD/`：momo 导出的原始 TXT 和图片，**不要动**。
- 不要移动、重命名或删除原始导出文件夹，除非用户明确说可以。

## 3. 通用工作原则

- **隐私优先**：日记内容高度私人。未经用户允许，不得把内容摘要、引用或分析结果发到外部。
- **谨慎破坏性操作**：批量删除、覆盖、格式化前必须确认。
- **不要自动提交/推送**：仓库如果是 git 仓库，也不要自动 commit/push，除非用户明确说可以。
- **使用中文回复**：用户主要使用中文交流。
- **保持简洁**：规则、脚本和输出都不要过度工程化。

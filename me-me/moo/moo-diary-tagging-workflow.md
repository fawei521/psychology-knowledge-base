- `心理学 / 阅读 / 学习` — 读书、心理学理论/概念、课堂笔记、学习方法、mbti、摘抄
- `关系 / 爱情 / 友情` — 恋爱、友情、人际、社交、被拒绝、告别

- `家庭 / 原生家庭` — 父母、姐弟、童年、家庭循环、亲情

### `genre`

- `诗歌` — 分行明显、押韵、意象集中、短句抒情
- `散文` — 抒情或议论段落
- `叙事` — 记录事件、日常、清单、过程
- `摘抄` — 读书笔记、概念整理、课堂笔记
- `引用` — 单独一句名言/诗句/台词，无个人展开
- `对话/交流` — 明显的自我对话、第二人称、问答体
- `混合` — 难以归入单一类型

### `origin`

- `原创` — 绝大多数
- `引用` — 单独引用他人文字（诗句、名言、台词）
- `摘抄` — 大段整理他人内容
- `AI生成` — 明确由 AI 生成且作为主要内容

## 判断优先级

1. 优先看 **标题 + 正文前 300 字**。
2. `category` 以内容主题为准，不要只看 `activity`。
3. 恋爱/失恋/人际关系优先入 `关系 / 爱情 / 友情`，即使夹杂自我反思。
4. 纯心理学概念/读书笔记优先入 `心理学 / 阅读 / 学习`。
5. 明显引用且几乎无个人发挥的，genre 用 `引用`，origin 用 `引用`。
6. 有心理概念但做了自己整理/笔记的，genre 用 `摘抄`，origin 用 `原创`。
7. 空内容或仅有标题的，按标题倾向给；无法判断则 `日常 / 生活片段` + `散文` + `原创`。

## 操作流程（批量）

1. 读取 `E:\me me\moo\.temp\incomplete_list.txt` 获取剩余未加标签文件清单。
2. 用脚本生成 `previews.json`：每篇提取 title / date / weather / mood / activity / 正文前 300 字。
3. 按 60 篇为一批读取预览，人工分类后写入 `classifications_batchN.json`。
4. 用脚本统一将分类写入各文件 frontmatter，保持字段顺序：
   `title` → `date` → `weather` → `mood` → `activity` → `category` → `genre` → `origin`。
5. 每批写完后更新进度，避免重复处理。
